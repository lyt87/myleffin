# 예상 소진일 계산기

기본 계산은 `index.html`을 브라우저에서 열어 사용할 수 있습니다.

네이버 검색추이 그래프까지 사용하려면 로컬 서버로 실행합니다.

```powershell
$env:NAVER_CLIENT_ID="네이버_클라이언트_ID"
$env:NAVER_CLIENT_SECRET="네이버_클라이언트_SECRET"
& "C:\Users\yongt\.cache\codex-runtimes\codex-primary-runtime\dependencies\node\bin\node.exe" server.js
```

실행 후 브라우저에서 `http://localhost:5173`을 엽니다.

## 계산 방식

- 일평균 판매량 = 판매량 파일의 판매량 합계 / 판매량 기준 기간
- 잔여 가능 수량 = 재고 + 미입고 + 리오더 수량 - 미발수량
- 예상 소진일 = 기준일 + 잔여 가능 수량 / 일평균 판매량

## 엑셀 컬럼

각 파일은 첫 번째 시트만 읽습니다. 컬럼명은 아래 이름 중 하나와 비슷하면 자동 인식합니다.

- 판매량 파일: `바코드`, `상품명(확정)`, `수량`
- 재고 파일: `품목코드`, `품목명[규격]`, `재고수량`
- 미입고 파일: `1-1. STYLE NO(라벨출력용)`, `9.컬러`, `10.사이즈`, `미구매수량`
- 미발수량 파일: `바코드`, `상품명(확정)`, `수량`

추가로 아래 컬럼명도 자동 인식합니다.

- 상품 식별: 상품코드, 품목코드, 품번, SKU, 상품번호, 바코드, 코드, 옵션코드, 제품코드
- 상품명: 상품명, 제품명, 품명, 옵션명, 상품명(확정), 품목명[규격], 품목명(규격)
- 판매량: 판매량, 주문수량, 출고수량, 판매수량, 수량
- 재고: 재고, 재고수량, 현재고, 가용재고, 수량
- 미입고: 미입고, 입고예정, 입고예정수량, 발주수량, 미입고수량, 미구매수량, 수량
- 미발수량: 미발수량, 미발, 미출고, 미배송, 미발주문, 수량

## 결과

결과표는 위험 상품이 먼저 보이도록 소진까지 남은 일수 순서로 정렬됩니다. 상품명 키워드로 카테고리를 자동 분류하며, 상품코드 옆에 컬러와 사이즈를 표시합니다.

`리오더 수량`을 입력하면 잔여 가능 수량과 예상 소진일 계산에 포함됩니다.

- 잔여 가능 수량 = 재고 + 미입고 + 리오더 수량 - 미발수량
- `리오더 추천`은 위험/주의 상품에 대해 네이버 검색추이 최근 30일 평균과 1년 평균을 비교해 추천 수량을 입력합니다.
- `리오더 엑셀 다운로드`는 리오더 수량을 입력한 상품만 `.xlsx`로 저장합니다.
- `CSV 다운로드`는 전체 계산 결과를 저장합니다.

카테고리를 클릭하면 최근 1년 네이버 검색추이를 그래프로 표시합니다. 이 기능은 네이버 DataLab API 키가 설정된 로컬 서버에서만 동작합니다.
